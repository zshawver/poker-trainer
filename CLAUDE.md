# Project: Poker Trainer

## What this project does
Backend for a poker math training program. Provides a fast, NumPy-vectorized 7-card hand evaluator (~2M hands/sec), an equity engine with a precomputed 169×169 preflop lookup table, an interactive game loop, and an archetype-driven villain bot. A FastAPI/React layer is scaffolded on top for a future web UI.

## Current status
Early development. The vectorized engine and game/villain logic were just migrated into the fullstack template. No API endpoints are wired to the engine yet; the FastAPI app currently has only the template's stub auth router and health check.

## Stack
- **Frontend:** React + TypeScript, Vite, Tailwind CSS, Zustand (template scaffolding, no UI yet)
- **Backend:** Python 3.11, NumPy, FastAPI, SQLAlchemy (async), Alembic, JWT auth
- **Infrastructure:** PostgreSQL 16, Redis 7, Docker Compose

## How to run
See `README.md`. Local backend dev: `conda env create -f backend/environment.yml`, `conda activate poker_trainer`, then run scripts or `pytest` from `backend/`.

## Folder structure notes
- `backend/src/engine/` — pure-math poker engine: `cards`, `evaluator`, `equity`, `precompute_equity`, `positions`, `ranges`, `variables`, plus the `preflop_equity.json` lookup table. Sibling modules use relative imports.
- `backend/src/services/` — game orchestration: `game.py` (PokerGame, HandState) and `villain.py` (5-archetype bot). Imports from the engine via `src.engine.*`.
- `backend/src/{api,core,db,models,schemas}/` — FastAPI scaffolding from the template; not yet wired to the engine.
- `backend/tests/test_evaluator.py` — pytest suite for the evaluator (edge cases, tiebreakers, ties, throughput benchmark).
- `frontend/` — React SPA scaffolding from the template; no poker UI yet.

## Key decisions
- Hand evaluation is fully vectorized over `(N, 7)` NumPy arrays. The hot path has no Python loops over hands; all operations are array broadcasts.
- Preflop range-vs-range equity is short-circuited with a precomputed 169×169 JSON table generated by `engine/precompute_equity.py`.
- The villain bot is intentionally non-vectorized (one decision at a time) but stays fast by using the cached equity lookups.
